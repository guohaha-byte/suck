# 光照监测系统 操作流程

普中麒麟 F407 开发板：光敏 ADC 采集 + HX8357DN 触摸屏显示 + ESP8266/MQTT(JSON) 上报与双 LED 控制。

---

## 一、准备工作

### 硬件
- 普中麒麟 STM32F407ZGT6 开发板
- 3.5" HX8357DN 彩屏（插到板上 LCD 接口，FSMC 并口）
- ESP8266 WiFi 模块（插到板上 ESP8266 接口，走 USART3）
- 板载光敏电阻 LS1（PF8 / ADC3_IN6，无需额外接线）
- ST-Link / J-Link 下载器，USB 线

### 软件
- Keil MDK-ARM 5.x（已装 STM32F4xx 器件包）
- 一个 MQTT 服务器：EMQX（公网或本机 `emqx` 服务，默认端口 1883）
- MQTT 调试工具：MQTTX（或 EMQX Dashboard）

---

## 二、获取代码

```bash
git clone <仓库地址>
cd suck
git checkout claude/admiring-pascal-k2oo3j
```

> 工程通过 `2.zip` 分发：解压 `2.zip` 得到 Keil 工程，再把分支中的
> `APP/adc`、`APP/lcd`、`APP/gui`、`APP/mqtt` 与 `User/main.c`、`Template.uvprojx`
> 覆盖进去即可（分支里已是覆盖好的完整树，直接用分支内容也行）。

## 三、填写配置（必做）

打开 `User/main.c`，修改顶部 4 个宏：

```c
#define SSID          "你的WiFi名"
#define PASD          "你的WiFi密码"
#define MQTTSV        "192.168.1.100"   // EMQX 服务器 IP
#define MQTTSV_PORT   "1883"            // EMQX 端口
```

主题（可选，默认即可）：
```c
#define TOPIC_PUB     "mcu"   // 设备 -> 服务器：上报光照
#define TOPIC_SUB     "app"   // 服务器 -> 设备：下发 LED 指令
```

> MQTT 登录账号在 `MqttDataDL[]` 里（ClientID `1014238195`、user `555798`、pass `mcu`）。
> 若 EMQX 开启了认证，请改成你自己的账号（或在 EMQX 关闭匿名限制）。

### 核对背光引脚（如屏不亮再看）
`APP/lcd/lcd.h` 中默认背光 = PB0：
```c
#define LCD_BL_PORT  GPIOB
#define LCD_BL_PIN   GPIO_Pin_0
```
若你的板子 `LCD_BL` 接别的脚，改这两行即可。

---

## 四、编译下载

1. 双击 `Template.uvprojx` 打开 Keil。
2. 选好下载器：**Options for Target → Debug** 选 ST-Link/J-Link，**Flash Download** 勾选 *Reset and Run*。
3. **Rebuild**（F7），应 0 Error。
4. **Download**（F8）烧录到板子。

> 工程已登记新文件与头文件路径；`stm32f4xx_fsmc.c`、`stm32f4xx_adc.c` 本就在工程内，无需手动加。

## 五、上电自检

上电后屏幕应显示：
```
        Light Monitor System
 Light :  ──
 LED1  :  OFF
 LED2  :  OFF
 WiFi  :  Disconnected
 MQTT  :  Disconnected
```
- 连上 WiFi 后 `WiFi` 变绿 `Connected`；
- 登录并订阅成功后 `MQTT` 变绿 `Connected`；
- `Light` 数值随光线变化（手电照→变大，遮挡→变小）。

> 调试串口：USART1（115200, 8N1）会打印 ESP8266 的 AT 应答，便于排查连接问题。

## 六、MQTT 联调

### 1. 看上报（设备 → 服务器）
用 MQTTX 连接同一个 EMQX，**订阅** 主题 `mcu`，应每 **3 秒**收到一帧 JSON：
```json
{"light":78,"led1":0,"led2":1}
```

### 2. 下发指令（服务器 → 设备）
用 MQTTX **发布**到主题 `app`，payload 为 JSON：

| 目的 | 发送内容 |
|---|---|
| 点亮 LED1 | `{"led1":1}` |
| 熄灭 LED1 | `{"led1":0}` |
| 点亮 LED2 | `{"led2":1}` |
| 同时控制 | `{"led1":1,"led2":0}` |

发送后：板上对应 LED 立即亮/灭，屏幕 `LED1/LED2` 状态同步刷新（绿=ON，灰=OFF），
下一帧上报的 JSON 里 `led1/led2` 也会反映新状态。

## 七、常见问题

| 现象 | 排查 |
|---|---|
| 屏白屏/不亮 | 确认 HX8357DN 插好；核对 `LCD_BL` 背光引脚；检查 FSMC 接线 |
| 屏花屏/偏色 | 多为时序，适当增大 `lcd.c` 中 `writeTiming/readTiming` 的 setup 值 |
| WiFi 连不上 | 看 USART1 打印；确认是 2.4G WiFi；SSID/密码无误 |
| MQTT 连不上 | 确认 EMQX IP/端口可达；账号是否需认证；防火墙 1883 |
| 收不到上报 | 订阅主题是否为 `mcu`；设备 MQTT 是否已 Connected |
| 下发不生效 | 发布主题须为 `app`；payload 是 `{"led1":1}` 这种 JSON |
| 光照数值反了 | `adc.c` 的 `Lsens_Get_Val()` 改成不取反：`return adc*100/4095;` |

## 八、数据流总览

```
 光敏 LS1 ──ADC3_IN6(PF8)──> STM32 ──每3s JSON──> ESP8266 ──WiFi──> EMQX ──> MQTTX
                                  │                                         │
 屏幕(HX8357 FSMC) <── 刷新 ──────┤                                  发布{"led1":1}
                                  │                                         │
 LED1/LED2(PF9/PF10) <── 控制 ────┴────── 解析JSON <── USART3 <── ESP8266 <─┘
```
